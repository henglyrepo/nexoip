# Security Policy

## Supported Versions

Security updates are provided for the latest version on the default branch.

## Reporting a Vulnerability

If you believe you have found a security vulnerability:

1. Do not open a public issue with proof-of-concept details.
2. Prefer GitHub's private vulnerability reporting (Security Advisories) for this repository.
3. If private reporting is not available, open a minimal public issue that says "security report" and we will follow up for details.

Please include:

- A description of the issue and potential impact
- Steps to reproduce
- Affected endpoints/pages
- Any relevant logs (redact tokens/credentials)
